# Contributing to JobTracker

Thanks for contributing to JobTracker 🚀

This project focuses on building a clean, maintainable, and modern application tracking experience while following practical frontend engineering principles.

---

## Development Workflow

### 1. Create an Issue

Before starting work:

- Create a GitHub issue
- Clearly define the goal
- Keep scope focused
- Add appropriate labels

Example labels:

- `feature`
- `refactor`
- `bug`
- `documentation`
- `enhancement`

---

## Branch Naming Convention

Use descriptive branch names:

```bash
feature/company-metadata-form
refactor/shared-constants
fix/modal-scroll-overflow
chore/update-seed-data
```

---

## Commit Convention

Use clear and focused commit messages.

Examples:

```bash
feat: add company metadata support
refactor: centralize shared constants
fix: resolve modal overflow issue
chore: update seed applications
```

Avoid:

```bash
update stuff
fix things
final changes
```

---

## Pull Request Guidelines

Each PR should:

- Solve a single problem
- Stay reasonably small
- Include a clear summary
- Link related issues
- Avoid unrelated changes

### PR Structure

```md
## Summary
Short explanation of the change.

## Changes
- Added...
- Updated...
- Refactored...

## Notes
Additional implementation details.

## Linked Issues
Closes #issue-number
```

---

## Code Style

### General Principles

- Prefer readable code over clever code
- Keep components focused
- Avoid premature abstraction
- Follow KISS and YAGNI principles
- Reuse constants when appropriate

### UI Principles

- Maintain consistent spacing
- Preserve visual hierarchy
- Keep interactions predictable
- Avoid cluttered layouts

---

## Before Submitting

Verify:

- Application builds successfully
- No console errors
- CRUD workflows still function
- Responsive layout still works
- No visual regressions introduced

---

## Project Philosophy

JobTracker prioritizes:

- Maintainable architecture
- Clean UI/UX
- Incremental improvements
- Practical frontend engineering
- Consistent Git workflows
